Functions

Functions always take in a predicate, and some take in a field as well.

avg({predicate}, {field})

The avg function takes in a predicate and a field. It finds the average of the field, taking into consideration the predicate query.

count({predicate})

The count function takes in a predicate and counts the number of records that satisfy the predicate's condition.

max({predicate}, {field})

The max function takes in a predicate and a field. It finds the maximum value of the field, taking into consideration the predicate query.

min({predicate}, {field})

The min function takes in a predicate and a field. It finds the minimum value of the field, taking into consideration the predicate query.

sum({predicate}, {field})

The sum function takes in a predicate and a field. It finds the sum of all the values of the field, taking into consideration the predicate query.

Fields

Fields are always prefixed with an "@" symbol.

@revenue

The @revenue field finds the total revenue for a given predicate.

@quantity

The @quantity field finds the total quantity for a given predicate. Quantities should be entered as integers.

@created_at

The @created_at field finds the records that were created at, before or after a certain date. Dates should be entered in the form of strings in the following format: "2016-06-01"

Example Functions

sum(@quantity > 1000, @revenue)

Sum the field revenue for all entries where the field quantity is greater than 1000.

count(@created_at > "2016-06-01")

Count the entries that were created after June 1, 2016.
